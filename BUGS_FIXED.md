# Исправленные баги

## ✅ 1. Баг с combatGap при leaderId === 0

**Проблема:**
Когда все игроки одного уровня (`leaderId === 0`), функция `applyLevelBias` делала ранний `return` и пропускала блок с `combatGap`. Контекст боя не применялся.

**Решение:**
- Убран ранний `return` для `leaderId === 0`
- Блоки с модификаторами лидера/не-лидера обёрнуты в `if (leaderId !== 0)`
- Блок с `contextModified` (контекст боя) теперь применяется ВСЕГДА, независимо от наличия лидера

**Результат:**
Теперь даже если все игроки одного уровня, контекст боя работает:
- При `combatGap >= 4` боты реже помогают игроку (25% шанс нейтрализации)
- При `combatGap <= -4` боты реже помогают монстру (40% шанс нейтрализации)

## ✅ 2. Баг с лишними игроками в resetMatch

**Проблема:**
После сброса матча в `playerLevels` оставались данные о старых игроках, если количество игроков было уменьшено.

Пример:
- Было 4 игрока
- Установили 2 игрока
- Сбросили матч
- В `playerLevels` остались записи для игроков 3 и 4

**Решение:**
Добавлена очистка лишних игроков после сброса:
```javascript
// Удаляем лишних игроков из playerLevels
Object.keys(playerLevels).forEach(id => {
    if (parseInt(id) > playerCount) {
        delete playerLevels[id];
    }
});
```

**Результат:**
После сброса матча в `playerLevels` остаются только текущие игроки.
